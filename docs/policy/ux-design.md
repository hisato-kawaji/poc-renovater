# UX Design & User Experience Principles

- 最終更新: 2026-06-30
- ステータス: Draft
- 対象読者: UI/UXデザイナー、フロントエンド開発者、プロダクトマネージャー

---

## 1. プロダクトビジョンとユーザー体験のゴール

PoC Foundryは、「単なるLLMチャット」ではなく、「アップロードされたコードベースを理解し、批判的レビューを交えながら自律的に改善していくインテリジェントな工場（Foundry）」である。
UXの最大のゴールは **「ユーザーがAIの作業プロセスを透明性をもって監視でき、必要に応じて介入できる『共同作業』の体験を提供すること」** である。

---

## 2. ターゲットペルソナ

1. **Tech Lead / ソフトウェアエンジニア**
   - **目的**: チームのプロトタイプコードやPoCを本番稼働可能な品質に引き上げたい。
   - **課題**: セキュリティ、クラウドインフラ、CI/CDの整備など、本質的でない作業に時間を取られたくない。
   - **ニーズ**: AIの出したPRをサクッとレビューしてマージしたい。

2. **プロダクトマネージャー / 起業家**
   - **目的**: MVPを素早く市場に出したい。
   - **課題**: 技術的な詳細（どのクラウドサービスを使うべきか等）が分からない。
   - **ニーズ**: AIと壁打ち（Charterフェーズ）することで要件を詰め、あとは勝手に動くアプリを作ってほしい。

---

## 3. カスタマージャーニー (As-Is / To-Be)

### Phase 1: Ingestion & Upload
- **体験**: ZIPファイルをドラッグ＆ドロップするだけで、数秒でAIがコードの中身を「理解」し、使われている技術スタックや問題点をリストアップする。
- **UI要件**: 直感的なドロップゾーンと、パース中のプログレス表示。解析結果はダッシュボードとして分かりやすく提示。

### Phase 2: Chartering (要件定義)
- **体験**: AIとのチャットUIを通じて、「このアプリは社内用か、一般公開用か」「データはどう保存すべきか」をインタラクティブに決定する。
- **UI要件**: 採点スコアがリアルタイムに変動するプログレスバー。閾値を超えると「Proceed to Next Phase」ボタンが有効になるゲーミフィケーション要素。

### Phase 3: Planning & Implementation (自律実行)
- **体験**: ユーザーはダッシュボードで、AIが現在どのIssueを作成し、どのブランチでコードを書き、PRを作成しているかをリアルタイムに監視する。
- **UI要件**: かんばんボード（Kanban）風、もしくはタイムライン風の進行状況表示。Live state（Firestore listener）を用いた遅延のないアップデート。

### Phase 4: Review & Deploy
- **体験**: 作成されたPRの差分を確認し、プレビュー環境のURL（Cloud Run）をクリックして実際の動作を確かめる。問題があればAIにフィードバックを返し、良ければ承認する。
- **UI要件**: 差分（Diff）ビュー、プレビューURLのハイライト表示、AIへのフィードバックコメント入力欄。

## 4. UI/UX デザイン原則

### 1. Transparency (透明性)
AIが「今何をしているのか」「なぜそのコード変更を行ったのか」を隠さない。ブラックボックス化を避け、Issueの分割理由やCharterスコアの根拠を明示する。

### 2. Controllability (制御性)
ユーザーはいつでも処理を停止（Kill Switch）でき、AIの提案を却下して手動で方針を修正できる。自動進行と手動介入のバランスを保つ。

### 3. Graceful Degradation (フェイルセーフ)
AIがコード生成に失敗したり、CIが落ちたりした場合、エラーを隠さず「何が問題で、どう回復するか（Self-Improvement）」のプロセスを提示する。

### 4. Professional & Clean (プロフェッショナルな美しさ)
shadcn/ui と Tailwind CSS を基盤に、洗練されたダッシュボードを提供する。B2B SaaS のような信頼感のあるタイポグラフィとカラースキームを採用する。

---

## 5. 主要画面のワイヤーフレーム概念

### A. グローバルダッシュボード (Multi-PoC View)
- 左サイドバー: テナント切り替え、PoC一覧、設定。
- メイン領域: 進行中のPoCの全体ステータス（カード形式）。各カードには「現在のフェーズ」「エラー有無」「最終更新」を表示。

### B. PoC詳細画面 (Agent Workspace)
- ヘッダー: PoC名、ステータスバッジ（例: `IMPLEMENTING`, `PR_OPEN`）、プレビューURL（デプロイ完了時）。
- タブ1: **Overview** (Charterのスコアと要件、解析結果)
- タブ2: **Timeline / Pipeline** (Issueの進捗、現在の処理ステップのLive表示)
- タブ3: **Pull Requests** (レビュー待ちのPRリスト、Diffプレビューへのリンク)

### C. Charter チャットUI
- 右ペインにスコアと要件のサマリ。
- 左ペインにAIとのチャット。要件を満たす回答をすると、右ペインの項目が自動でチェックされていくインタラクション。
